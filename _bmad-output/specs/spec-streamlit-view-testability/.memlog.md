updated: 2026-07-15T23:11
---

- (decision) Use streamlit.testing.v1.AppTest (confirmed available, installed streamlit==1.58.0) instead of the helper-module-extraction approach the source deferred-work entries speculated about. AppTest runs the actual script in a sandboxed runtime and exposes assertable results (at.error[0].value, os.environ post-run), so app.py/views/*.py can be tested as-is with zero production code changes.
- (capability) CAP-1: app.py's GEMINI_API_KEY secrets-bridging (lines 25-36) is exercised by a real AppTest run of app.py, covering both the st.secrets-present and st.secrets-absent cases. Success: a test asserts os.environ["GEMINI_API_KEY"] is set after an AppTest run with st.secrets mocked to contain the key, and a second test asserts the existing env/.env value is left untouched when it's already set (env always wins per app.py's own logic).
- (capability) CAP-2: At least one automated regression test per view file (admin.py, intake.py, jobs_list.py) proves a caught exception's display text is produced by error_display_text, not bare str(exc) -- catching a future typo or partial revert at any of the 10 call sites. Success: for each of the 3 files, a test forces an exception with an empty str() at one call site and asserts via AppTest's at.error[0].value that the rendered message is non-empty and names the exception type (not blank).
- (constraint) Every AppTest-based test MUST run with the working directory changed to a pytest tmp_path (monkeypatch.chdir) containing freshly-seeded data/jobs.db, data/profile.db, data/sponsors.db at those exact hardcoded relative paths -- verified these are hardcoded module-level constants in views/admin.py:38, views/intake.py:41+43, views/jobs_list.py:24-25, register/cli.py:15, not parameterized. Tests must never point at or touch the real project data/ directory.
- (constraint) Seeding the tmp data/*.db files is just calling connect() once per DB module (jobs.db/resume.db/register.db all run CREATE TABLE IF NOT EXISTS on connect) -- no new schema-seeding helper needed, reuse existing connect() functions.
- (constraint) CAP-2 does not require covering all 10 individual error_display_text call sites -- one representative regression test per file (3 total) is sufficient to catch a systemic typo/revert, matching the proportionate scope of the original deferred-work concern.
- (direction) Non-goal: not extracting business logic out of app.py/views/*.py into separately-testable helper modules -- AppTest tests the scripts as they exist today, no production code refactor.
- (direction) Non-goal: not achieving full coverage of all 10 error-display call sites or every view page's full user flow -- this closes the regression-detection gap the deferred-work entries named, not a general Streamlit test suite.
