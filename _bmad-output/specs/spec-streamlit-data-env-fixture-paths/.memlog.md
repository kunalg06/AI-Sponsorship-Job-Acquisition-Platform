topic: Make streamlit_data_env return pre-built DB paths instead of forcing consumers to re-derive them
updated: 2026-07-16T23:46
---

- (note) Source: deferred-work.md entry under spec-streamlit-view-testability.md - tests/conftest.py's streamlit_data_env fixture (conftest.py:18-32) chdirs into tmp_path and seeds data/jobs.db, data/profile.db, data/sponsors.db, but returns bare tmp_path, forcing every consumer to re-derive tmp_path / 'data' / 'jobs.db'-style paths inline.
- (note) Verified current usage via grep: 4 real call sites re-derive the path inline - test_ui_actions.py:335,349 and test_views_error_display.py:66,99 - all identical streamlit_data_env / 'data' / 'jobs.db'. test_app.py's 3 usages never touch the returned value directly (fixture used only for its chdir+seed side effect). Confirms the reviewer's prediction ('before more test files start depending on it') already happened - real duplication now, not hypothetical.
- (decision) Return shape: a dict with keys root/jobs_db/profile_db/sponsors_db - matches this exact test suite's own established precedent (test_ui_actions.py's ui_tailor_env fixture, lines 213-258, already returns a dict with jobs_db/profile_db keys accessed the same bracket-syntax way). No new pattern introduced; resolves the reviewer's 'design choice worth deciding once' by reusing an existing local convention instead of inventing one.
- (capability) CAP-1 intent: streamlit_data_env returns ready-made paths for every DB it seeds (plus the sandbox root), so consumers use a lookup instead of re-deriving a data/<name>.db path inline. success: all 4 existing consumer call sites (test_ui_actions.py:335,349; test_views_error_display.py:66,99) use the fixture's returned lookup instead of path-joining, and every currently-passing test in these 3 files still passes unchanged in behavior.
- (constraint) Return shape is a dict keyed root/jobs_db/profile_db/sponsors_db, matching ui_tailor_env's existing precedent in this same test suite - not a dataclass/namedtuple, to avoid introducing a second competing pattern for the same kind of fixture.
- (constraint) The fixture's seeding behavior (chdir into tmp_path, create data/, connect+close each of the 3 DBs) is unchanged - only the return value's shape changes.
- (constraint) test_app.py's 3 usages need no changes - they only use the fixture for its side effect, never touching the returned value.
- (non-goal) No consolidation of streamlit_data_env and ui_tailor_env into one shared fixture - they seed different things (streamlit_data_env seeds all 3 empty DBs for AppTest page runs; ui_tailor_env seeds a specific job + profile + source docx for direct-call tests) and merging them is a bigger refactor than this fix.
- (non-goal) No change to what data/db paths actually resolve to on disk (still data/jobs.db etc under the chdir'd tmp_path) - purely how consumers access those same paths.

- (event) Self-validate Pass 2 (preservation): PASS.
