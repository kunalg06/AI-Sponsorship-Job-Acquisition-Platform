topic: Make the 'Regenerate tailored resume & cover letter' button actually regenerate instead of silently hitting the cache
updated: 2026-07-15T23:33

- (decision) No new UI control needed (checkbox/second button), contrary to the original deferred-work entry's framing. views/intake.py and views/jobs_list.py already compute already_generated to pick the button's own label ('Regenerate...' vs 'Generate...') - reusing that same boolean as the force argument makes the button's behavior match its label with zero new UI surface.
- (capability) CAP-1: Clicking the tailor button when it reads 'Regenerate tailored resume & cover letter' triggers an actual fresh regeneration (new Gemini call, docx files overwritten), not a cache hit. Success: a test calls generate_tailored_docx_for_job with force=True against a job whose docx cache is already valid and asserts the underlying LLM-calling path runs again (freshly_generated=True / the mock is invoked), not short-circuited by the cache check.
- (capability) CAP-2: Clicking the button when it reads 'Generate tailored resume & cover letter' (no cache yet) behaves exactly as today - unchanged. Success: existing tests covering the first-generation path still pass unmodified.
- (constraint) force=True bypasses the job_id-keyed cache-hash+file-existence check in jobs/cli.py's _get_or_generate_tailor_text, triggering a real Gemini API call and overwriting the existing resume/cover-letter docx files - a real cost, not a free no-op.
- (constraint) No confirmation dialog required before this regeneration - matches this codebase's existing convention that small-blast-radius single-click mutations (e.g. 'Mark discarded') don't need one; only bigger-blast-radius admin actions (whole-table register wipe, profile supersession) got confirmation dialogs in a prior spec. The button's own label already signals intent unambiguously.
- (direction) Non-goal: not adding any new UI element (checkbox, second button, confirmation dialog) - the fix is purely wiring the already-computed already_generated boolean through as the force argument.
- (direction) Non-goal: not changing the plain-text tailor command (jobs/cli.py's _cmd_tailor), which is already deliberately uncached/always-fresh and unrelated to this docx-path cache-check.
