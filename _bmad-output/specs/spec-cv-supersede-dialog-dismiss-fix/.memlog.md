updated: 2026-07-16T23:55

- (note) Source: deferred-work.md entry under spec-admin-destructive-safety.md - views/admin.py's _confirm_supersede_dialog (lines 142-156) can't be dismissed via the modal's native X/backdrop-close, only its in-dialog Cancel button clears pending_cv_profile/cv_registration_in_progress. Dismissing via X visually closes the modal, but the calling script re-invokes _confirm_supersede_dialog on the next rerun since it's gated on pending_cv_profile staying set (admin.py:208-210), so the modal reopens.
- (note) The original spec called this unfixable without a state-machine redesign, citing 'st.dialog has no documented callback for X/backdrop dismissal.' Verified against the actually-installed streamlit==1.58.0 (via help(st.dialog)) that this is now false: st.dialog accepts on_dismiss, which can be a callable - streamlit runs it as a callback (session_state accessible) before the rest of the script reruns, specifically on X/backdrop/ESC dismissal. Confirmed the sponsor-register-refresh dialog (_confirm_refresh_dialog) is unaffected - it's invoked directly from a button click, not gated on persisted session_state, so it doesn't reopen after dismissal (matches the original spec's own note).
- (capability) CAP-1 intent: dismissing the CV-upload supersede-profile dialog via X/backdrop/ESC clears the same pending state the in-dialog Cancel button already clears, so the dialog does not reopen on the next rerun. success: after a simulated X/backdrop/ESC dismissal (via streamlit's on_dismiss callback), st.session_state no longer has pending_cv_profile set and cv_registration_in_progress is False - the same end state Cancel already produces - verified via an AppTest-based test that triggers the dismiss path.
- (note) Checked streamlit.testing.v1.AppTest's public API (dir(at)) - no dismiss()/close-dialog method exists. X/backdrop/ESC dismissal is a browser-level interaction with no Python-level AppTest hook to simulate directly. Revising CAP-1's success criterion: test the extracted cleanup helper directly (unit-level - call it, assert session_state is cleared), not a full AppTest dismiss-simulation - the helper being the literal on_dismiss callback is what makes this reliable, not an end-to-end dismiss trigger.
- (capability) CAP-1 (revised) intent: dismissing the CV-upload supersede-profile dialog via X/backdrop/ESC clears the same pending state the in-dialog Cancel button already clears, so the dialog does not reopen on the next rerun. success: a shared cleanup helper is extracted and both (a) the Cancel button and (b) the dialog's on_dismiss parameter call it - verified by a direct unit test on the helper (seed session_state with pending_cv_profile set and cv_registration_in_progress=True, call the helper, assert both are cleared) plus a check that on_dismiss is wired to that exact helper (not a separate/diverged implementation).
- (constraint) Extract the Cancel button's existing cleanup (pop pending_cv_profile, set cv_registration_in_progress=False) into one shared helper function - both Cancel and on_dismiss call it, no duplicated logic.
- (constraint) Only _confirm_supersede_dialog changes. _confirm_refresh_dialog (sponsor-register refresh) is untouched - it's invoked directly from a button click, not gated on persisted state, so it has no equivalent bug.
- (constraint) Cancel's existing st.rerun() call stays Cancel-specific (on_dismiss already triggers its own rerun per streamlit's own dismiss-handling, calling st.rerun() again inside a callback is unnecessary/undefined) - only the state-clearing logic is shared, not the rerun call.
- (non-goal) No end-to-end/browser-level test of the actual X/backdrop/ESC click - AppTest's public API has no hook for simulating a dialog dismissal event (verified: no dismiss()/close method exists). Coverage is at the shared-helper unit level plus confirming the wiring, not a full interaction simulation.
- (non-goal) No broader state-machine redesign of the CV-upload flow - this is a narrow fix using the now-available on_dismiss parameter, not the bigger rework the original spec assumed was necessary.
